Ogma

  2. Pas lisible, pas facile a naviguer
  3. Plusieurs exploitations possible

Que faire d'un gros Dataset ?

  1. Exploration
  2. Visualisation globale
  3. Visualisation ciblée

Exploration

  1. Petit graph: on peut enqueter
  2. Gros graph: on ne voit plus rien

L'exploration requiers

  • Des requetes ciblées
  • Possibilité d'étendre les noeuds
  1. Limiter la taille du graph, points d'interets

Chargment

  • Charger par chunks
  • Charger intelligement
  1. Plus reactif, meilleur UX, plus rapide aussi
  2. Charger que le necessaire (topo, data), eveiter de filtrer

Charger par chunks


      // setGraph
      ogma.setGraph(myGiganticGraph1, {batchSize: 1000});
      // addGraph
      ogma.addGraph(myGiganticGraph2, {batchSize: 1000});
      // addNodes
      ogma.addNodes(myNumerousNodes, {batchSize: 1000});
      // addEdges
      ogma.addEdges(myNumerousEdges, {batchSize: 1000});
DEMO
  1. Toutes les fonctions ont l'option batchSize
